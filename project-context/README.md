# Project Context

This folder stores working context for BrahmiBhojan so decisions, progress, assumptions, and next steps remain visible across planning and implementation sessions.

It is not a replacement for formal architecture documents in `docs/`. Formal approved documents remain the source of truth. This folder is for operational memory and project continuity.

## Work Progress Tracking

Use `project-context/work-progress/` as the single place to track implementation execution.

- `task-log.md`: Add one entry after every completed task.
- `development-status.md`: Update module percentages and current phase.
- `todo-backlog.md`: Move tasks between Open and Done.
- `edge-cases-and-risks.md`: Capture edge scenarios and deferred risks.
